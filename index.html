PowerBuilder 簡介

PowerBuilder 是一個快速應用程式開發 (Rapid Application Development, RAD) 工具，專為開發企業級資料庫應用程式而設計。自從1991年推出以來，它已成為開發資料庫前端應用程式的重要工具。

PowerBuilder 的特點

  • 強大的圖形化開發介面，降低開發難度
  • 獨特的 DataWindow 技術，簡化資料庫操作
  • 支援物件導向程式設計，提高代碼可重用性
  • 低程式碼開發，提高生產力
  • 廣泛應用於企業管理系統、金融系統、報表系統等

適合的學習對象

  • 無程式開發經驗但希望進入開發領域的新手
  • 有其他程式語言經驗，希望學習資料庫應用程式開發的開發者
  • 需要維護或開發 PowerBuilder 應用系統的 IT 人員
  • 希望了解 Client/Server 架構應用程式開發的學習者

PowerBuilder 開發環境

PowerBuilder 提供了一個完整的整合開發環境 (IDE)，讓開發者能夠有效率地設計、編碼和測試應用程式。理解這個環境是學習 PowerBuilder 的第一步。

IDE 主要元件

  • 功能表列 (Menu Bar)：提供各種功能和操作選項
  • 工具列 (Toolbars)：包含常用功能的快捷按鈕
  • - PowerBar：用於打開 PowerBuilder 的各種畫板
  • - PainterBar：提供當前畫板常用操作的圖標按鈕
  • - StyleBar：用於調整文字的字體、字號等屬性
  • 系統樹 (System Tree)：顯示應用程式的物件階層結構
  • 畫板區 (Painter Area)：設計和編輯物件的工作區
  • 輸出視窗 (Output Window)：顯示編譯訊息和錯誤資訊

常用畫板 (Painters)

應用畫板 (Application)
設定應用程式屬性
窗口畫板 (Window)
設計使用者介面視窗
數據窗口畫板 (DataWindow)
建立資料顯示和編輯元件
函數畫板 (Function)
建立全局函數

環境設定步驟

  1. 下載並安裝 PowerBuilder 開發工具
  2. 設定資料庫連線參數
    SQLCA.DBMS = "SNC SQL Native Client(OLE DB)";

    SQLCA.Database = "YOUR_DATABASE_NAME";

    CONNECT USING SQLCA;
  3. 設定工作目錄和應用程式庫 (PBL)
  4. 自訂介面偏好設定

您的第一個 PowerBuilder 應用程式

讓我們一起建立一個簡單的 "Hello, PowerBuilder!" 應用程式，這將幫助您了解 PowerBuilder 開發的基本流程。

步驟 1：建立新的 Application 物件
  1. 在 PowerBuilder IDE 中，選擇 "File" → "New..." 選單
  2. 選擇 "Application" 物件類型
  3. 設定應用程式名稱為 "HelloApp"
  4. 點擊 "Save" 儲存應用程式
步驟 2：建立新的 Window 物件
  1. 選擇 "File" → "New..." 選單
  2. 選擇 "Window" 物件類型
  3. 將視窗的標題屬性設為 "我的第一個 PowerBuilder 應用程式"
  4. 從工具箱中拖曳一個 Button 控制項到視窗上
  5. 將按鈕的文字屬性設為 "點擊我"
  6. 將視窗命名為 "w_hello" 並儲存
步驟 3：編寫按鈕點擊事件
  1. 雙擊按鈕控制項，開啟 Script 編輯器
  2. 在 Clicked 事件中輸入以下程式碼：
MessageBox("Hello", "Hello, PowerBuilder!")
步驟 4：設定應用程式啟動視窗
  1. 開啟應用程式物件的 Script 編輯器
  2. 在 Open 事件中輸入以下程式碼：
Open(w_hello)
步驟 5：執行應用程式
  1. 按下 F5 鍵或點擊工具列中的 "Run" 按鈕
  2. 應用程式將啟動並顯示您設計的視窗
  3. 點擊按鈕，將顯示 "Hello, PowerBuilder!" 訊息框

理解應用程式結構

在 PowerBuilder 中，應用程式由多個物件組成，形成階層結構：

1. Application 物件

  • 應用程式的入口點
  • 控制應用程式的生命週期
  • 包含全域設定和資源

2. Window 物件

  • 使用者介面的基本單位
  • 包含各種控制項和視覺元素
  • 處理使用者互動和顯示邏輯

3. 控制項 (Controls)

  • 按鈕 (Button)、文字框 (EditMask)、標籤 (StaticText) 等
  • 每個控制項都有自己的屬性和事件
  • 通過事件處理程式回應使用者互動

重要概念: 事件驅動程式設計

在上面的例子中，我們可以看到事件驅動程式設計的工作方式：

  1. 使用者點擊按鈕
  2. 這觸發了按鈕的 Clicked 事件
  3. PowerBuilder 執行我們為該事件編寫的代碼
  4. 代碼顯示訊息框

DataWindow 元件介紹

DataWindow 是 PowerBuilder 最強大、最具特色的元件，它簡化了與資料庫的互動，並提供了豐富的資料展示和編輯功能。

DataWindow 的功能與優勢

  • 簡化資料存取：無需寫複雜的 SQL 語句即可檢索和操作資料
  • 彈性的資料呈現：支援多種顯示格式，包括表格、表單、圖表等
  • 內建資料驗證：提供客戶端資料驗證機制，確保資料正確性
  • 自動資料緩存：在本地緩存資料，減少與資料庫的交互
  • 增強的報表功能：可以輕鬆產生專業質量的報表
建立 DataWindow 物件的步驟
  1. 選擇 "File" → "New..." 選單
  2. 選擇 "DataWindow" 物件類型
  3. 選擇資料來源類型（例如 "SQL Select"）
  4. 編寫 SQL 查詢或使用查詢設計器
  5. 選擇顯示風格（例如 "Tabular", "Freeform", "Grid" 等）
  6. 自訂外觀和行為
  7. 命名並儲存 DataWindow 物件

在視窗中使用 DataWindow

  1. 在視窗設計器中，從工具箱拖曳 DataWindow 控制項到視窗上
  2. 設定控制項的 DataObject 屬性為您剛建立的 DataWindow 物件
  3. 編寫程式碼連接資料庫並檢索資料：
// 在視窗的 Open 事件中
dw_1.SetTransObject(SQLCA) // 設定交易物件
dw_1.Retrieve() // 檢索資料

DataWindow 的常用操作

資料檢索 (Retrieve)
從資料庫載入資料到 DataWindow 中
dw_1.Retrieve(param1, param2) // 帶參數的檢索
資料更新 (Update)
將變更儲存回資料庫
IF dw_1.Update() = 1 THEN
  COMMIT USING SQLCA;
ELSE
  ROLLBACK USING SQLCA;
END IF
資料篩選 (Filter)
只顯示符合條件的資料
dw_1.SetFilter("salary > 50000")
dw_1.Filter()
資料排序 (Sort)
根據特定欄位排序資料
dw_1.SetSort("last_name A, first_name A")
dw_1.Sort()

DataWindow 事件處理

DataWindow 提供多種事件讓您可以響應使用者互動和資料變更：

  • ItemChanged：當使用者修改資料時觸發
  • RowFocusChanged：當焦點移動到不同行時觸發
  • Clicked：當使用者點擊 DataWindow 時觸發
  • ButtonClicked：當點擊 DataWindow 中的按鈕時觸發
  • RetrieveStart/End：資料檢索開始/結束時觸發

PowerScript 語法基礎

PowerScript 是 PowerBuilder 的程式語言，用於實現應用程式的業務邏輯和互動功能。掌握 PowerScript 的基本語法是開發 PowerBuilder 應用程式的關鍵。

變數宣告與資料類型

PowerScript 支援多種資料類型，在使用變數前必須先宣告：

// 基本資料類型
Integer li_count = 0 // 整數
Long ll_id = 123456 // 長整數
String ls_name = "John" // 字串
Boolean lb_flag = True // 布林值
Date ld_today = Today() // 日期
Time lt_now = Now() // 時間
DateTime ldt_current // 日期時間
Decimal ldec_price = 19.99 // 小數

// 變數作用域前綴
// l_ 區域變數 (Local)
// i_ 實例變數 (Instance)
// g_ 全域變數 (Global)

PowerScript 變數命名慣例使用前綴來表示資料類型和作用域，這是良好的程式設計實踐。

運算符與表達式

PowerScript 支援常見的運算符：

算術運算符： + (加), - (減), * (乘), / (除), ^ (次方)
比較運算符： = (等於), <> (不等於), > (大於), < (小於)
邏輯運算符： AND, OR, NOT
// 運算符示例
Integer li_a = 10, li_b = 20
Integer li_sum = li_a + li_b // 加法
String ls_fname = "John"
String ls_lname = "Doe"
String ls_full = ls_fname + " " + ls_lname // 字串連接
Boolean lb_valid = li_a > 0 AND li_b < 100 // 邏輯運算
控制結構

PowerScript 提供多種控制結構來實現程式邏輯：

IF 條件判斷

IF condition THEN
  // 程式碼
ELSEIF another_condition THEN
  // 程式碼
ELSE
  // 程式碼
END IF

CHOOSE CASE 分支

CHOOSE CASE variable
  CASE value1
    // 程式碼
  CASE value2, value3
    // 程式碼
  CASE ELSE
    // 程式碼
END CHOOSE

FOR 迴圈

FOR i = 1 TO 10
  // 程式碼
NEXT

DO WHILE/UNTIL 迴圈

DO WHILE condition
  // 程式碼
LOOP

DO UNTIL condition
  // 程式碼
LOOP
函數與事件

PowerBuilder 中的函數和事件是實現程式邏輯的關鍵：

自訂函數

// 函數定義
FUNCTION ReturnType function_name (ArgumentType arg1, ArgumentType arg2)
  // 程式碼
  RETURN value
END FUNCTION

// 函數呼叫
result = function_name(param1, param2)

事件處理

// 事件處理程式
// 在控制項的 Clicked 事件中
MessageBox("通知", "按鈕被點擊了!")

// 觸發事件
cb_button.TriggerEvent(Clicked!)

// 延遲觸發事件
cb_button.PostEvent(Clicked!)

PowerScript 語法特色

物件導向程式設計

PowerScript 支援物件導向程式設計的核心特性：

  • 封裝：將資料和行為包裝在物件中
  • 繼承：透過繼承共享和擴展功能
  • 多型：同一操作在不同物件上有不同行為

代表詞的使用

PowerScript 提供特殊代表詞來參考物件：

  • This：代表當前物件
  • Parent：代表控制項所在的視窗
  • Super：用於存取父類別的功能

命名規範

PowerBuilder 開發中常用的命名規範：

  • 變數名稱使用前綴表示資料類型和作用域
  • 控制項名稱使用前綴表示類型（如 cb_button 表示 CommandButton）
  • 視窗名稱通常以 w_ 開頭
  • DataWindow 物件名稱通常以 d_ 開頭

注意事項

編寫 PowerScript 時的一些重要注意事項：

  • PowerScript 不區分大小寫
  • 使用 `//` 添加單行注釋，使用 `/* */` 添加多行注釋
  • 使用 `&` 符號在程式碼行尾表示續行
  • 使用 `IsNull()` 函數檢查變數是否為空
  • 使用 `TRY-CATCH` 區塊處理異常

常見問題

PowerBuilder 是一個快速應用程式開發 (RAD) 工具，特別適合開發資料庫前端應用程式。它最適合用於企業內部管理系統、金融系統、庫存管理、客戶關係管理 (CRM) 系統等需要強大資料處理能力的應用程式。PowerBuilder 的 DataWindow 技術使其在資料庫操作方面特別高效。

學習 PowerBuilder 不需要太多前置知識，但具備以下背景會更有幫助：

  • 基本的程式設計概念（變數、函數、控制結構等）
  • SQL 和關聯式資料庫的基本知識
  • 物件導向程式設計的基本概念
  • 圖形化使用者介面 (GUI) 的基本理解

但即使您是完全的初學者，只要循序漸進地學習，也能掌握 PowerBuilder 開發。

PowerBuilder 相比其他程式語言有以下優勢：

  • DataWindow 技術：提供了強大且靈活的資料庫存取和顯示功能
  • 快速開發：低程式碼的方式能快速建立功能性強大的應用程式
  • 易於學習：相較於其他程式語言，PowerBuilder 的學習曲線較平緩
  • 整合性：提供完整的開發環境，從設計到部署一應俱全
  • 成熟度：經過多年發展，擁有穩定性高的技術和豐富的程式庫

雖然 PowerBuilder 不是最新的技術，但它仍然在全球多個行業中廣泛使用，特別是在金融、保險、政府機構和大型企業中。許多使用 PowerBuilder 開發的核心業務系統仍在運行，並需要維護和更新。

學習 PowerBuilder 的價值主要體現在：

  • 維護現有系統的工作機會
  • 對資料庫應用程式開發原理的深入理解
  • 特定行業的專業知識積累
  • 較少競爭的特定技能市場

此外，PowerBuilder 仍在不斷更新，新版本支援 .NET 和 Web 開發，顯示了該技術的延續性。
